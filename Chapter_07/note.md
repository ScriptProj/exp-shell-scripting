#### ch6 设置变量和默认值  
##### 6.1 设置默认值  
```bash
#检查变量是否被设置 
[ -z "$VAR" ] && VAR="default value"
[ "$VAR" ] || VAR="default value"
```


##### 6.2 变量替换  
```bash
# 开始的冒号是一个正确执行非活动任务的shell命令,它会扩展行中紧随其后的所有参数
# :=句法表示VAR变量将会和"some default"字符串进行比较
# 如果VAR还没有被设置,将:=之后表达式的值赋给它
: ${VAR:="some default"}
# 可在一行中给多个变量设置默认值
: ${VAR:="some default"} ${VAR2:=42} ${VAR3:=$LOGNAME}
# 大多数情况下,代码中执行替换句法的地方,这些替换仅仅用某个值替换了变量,但并没有设置变量,就是说变量并没有被真正赋值


# =句法
username=""
echo "${username=$LOGNAME}"
# 在这个句法中,命令将会输出"echo"之后语句的执行结果.因为变量username虽然为空值,但已经被定义了,所以除了一个回车不会再有其他输出
# 只有当username变量完全没有被定义时,才会将其设置为变量LOGNAME的值
# 当脚本或函数需要依赖于某些定义变量时,就要使用这种句法.它主要应用于登录或cron脚本中.
```
